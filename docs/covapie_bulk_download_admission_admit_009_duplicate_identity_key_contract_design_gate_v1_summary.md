# CovaPIE ADMIT_009 duplicate identity key contract design gate v1

This metadata-only gate freezes `duplicate_identity_key` as an exact built-in `str` matching `^covapie_dup_v1_sha256_[0-9a-f]{64}$`. The canonical value is the input unchanged: no coercion, trimming, case conversion, Unicode normalization, aliasing, repair, digest recomputation, or regeneration is allowed. The evaluator-facing key is atomic and opaque with zero visible chemical components; its canonical descriptor, deterministic SHA256 assignment, versioning, and fail-closed digest-collision handling remain producer responsibilities.

The policy item is an exact built-in `str` equal to `covapie_duplicate_identity_key_contract_v1`. `batch_duplicate_identity_keys` is an exact built-in `tuple`; every member is an exact fully valid key, and members are already strictly ASCII-string ascending and unique. The caller supplies the snapshot of earlier admitted or queued candidates and excludes the current record. Exact case-sensitive string membership alone decides unique (`passed`) versus duplicate (`blocked`, `DUPLICATE_IDENTITY_KEY_ALREADY_PRESENT`). Scalar, policy, and batch failures short-circuit in that order and produce `invalid`; policy- and batch-invalid paths retain the canonical candidate pair, while scalar-invalid paths do not. The unified result invariant is exact: `passed` iff outcome is `passed`, and `blocks_candidate` iff outcome is not `passed`; therefore every invalid or blocked result blocks the candidate, while a passed result does not.
