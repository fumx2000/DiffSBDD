# CovaPIE unified dispatch runtime with ADMIT_001–014 v1

## Scope

This successor adds the `ADMIT_014` in-memory unified adapter, immutable
Exact14 registry, and single-rule dispatcher on top of the committed
ADMIT_001–013 runtime. It does not alter predecessor files, register
`ADMIT_015`, aggregate rules, enforce pre-download authorization, map a
provider, access raw data, execute a network/download operation, or touch
model, checkpoint, dataloader, or training code.

The public runtime closure ends at:

```text
# === CovaPIE ADMIT_001 TO ADMIT_014 PUBLIC RUNTIME CLOSURE END ===
```

Before that marker, the module imports only the Exact13 predecessor, the
committed ADMIT_014 standalone evaluator, the committed independent formal
interface oracle, and Python in-memory type helpers. The adapter-design
simulator is imported only after the marker for evidence construction.

## Runtime contract

- The shared Exact13 result type, dispatch error type, fields, codes, and
  vocabularies are exact predecessor object re-exports.
- The first thirteen registry handler values preserve exact object identity.
- `EVALUATOR_REGISTRY`, `RULE_NAMES`, and `ADAPTER_IDS` are immutable
  `MappingProxyType` objects ordered `ADMIT_001` through `ADMIT_014`.
- `ADMIT_015` remains known but is neither callable, adapter-ready, nor
  registered.
- The public dispatcher has the predecessor signature and precedence and
  evaluates exactly one rule through the successor-local registry.

The ADMIT_014 handler validates forbidden contexts before the candidate,
never reads a valid candidate mapping, forwards the identical stage context
to the formal evaluator and oracle, calls each exactly once after its
preceding gate succeeds, attests the formal and oracle Exact9 values, requires
field-by-field exact type/value equivalence, and projects only a validated
Exact9 value to the shared Exact13 result. Stage booleans project to lowercase
`"false"` or `"true"`; stage fields never leak into candidate fields.
