# CovaPIE unified dispatch runtime with ADMIT_001 to ADMIT_007 v1

This successor implements the committed `ADMIT_007` unified adapter contract and extends the immutable single-rule dispatch registry from Exact6 to Exact7. It reuses the predecessor's public result and dispatch-error types, schema constants, and `evaluate_admission_rule` signature by identity. The six `ADMIT_001` through `ADMIT_006` handlers are also reused by identity; only `_evaluate_registered_admit_007` is new.

The public dispatch order remains exact built-in string validation, known-rule validation, registration validation, and handler invocation. `ADMIT_008` through `ADMIT_015` remain known but not registered and fail closed. This runtime exposes neither `evaluate_all_rules` nor a combined candidate verdict.

The `ADMIT_007` handler enforces the frozen context order before any candidate access: `batch_context` must be `None`, `evaluation_context` must be a `Mapping` containing `allowed_covalent_evidence_classes`, and download-result and stage-authorization contexts must be `None`. The required context value is obtained with exactly one `__getitem__` lookup before the download and stage gates. The original routed value is passed without adapter-side semantic prevalidation or copying.

Candidate mappings and their subclasses are accepted without copying or mutation. The required candidate value is also obtained with exactly one `__getitem__` lookup. A non-mapping candidate projects directly to Exact13 invalid. A missing key, exact `None`, or exact built-in empty string projects to `covalent_event_evidence_missing` without evaluator or oracle calls. Other values, including empty string subclasses, whitespace, malformed objects, and `distance_only_inference`, reach the standalone evaluator unchanged.

Executable candidates call `evaluate_admit_007` exactly once with both original objects. Before the independent enum oracle can run, the handler requires the exact committed `Admit007EvaluationResult` type, requires `vars(source)` to be an exact `dict` whose ordered keys are the committed Exact10 fields, verifies the dataclass field shape, reconstructs the committed dataclass, and checks complete invariants. Missing or extra instance storage is rejected before the oracle. It then calls `classify_admit_006_admit_007_evidence_design` exactly once with the same objects and requires full Exact10 equality. Any type, invariant, or oracle mismatch fails closed as adapter not ready and produces no Exact13 projection.

Valid Exact10 results project to the predecessor's Exact13 result without added normalization. Explicit evidence remains passed, `distance_only_inference` remains blocked with `DISTANCE_ONLY_INFERENCE_NOT_ADMISSIBLE`, scalar-invalid results retain empty canonical state, and context-invalid results with a canonical scalar retain their canonical validated and normalized pair.

The deterministic evidence contains 56 runtime-contract rows, 107 truth rows, 15 registry rows, 36 safety rows, and the successor Exact11 issue inventory. Truth groups are `global_dispatch` 5, `predecessor_identity_and_behavior` 14, `context_routing` 7, `candidate_envelope` 12, `standalone_semantics` 37, `source_fail_closed` 15, `projection_semantics` 7, and `unsupported_boundary` 10. Only `UNIFIED_ADMISSION_RULE_COVERAGE_INCOMPLETE` changes from the authoritative ADMIT_007 adapter-design issue inventory: it remains open but now covers only `ADMIT_008` through `ADMIT_015`.

The metadata gate uses a fixed ordered Exact18 committed-source boundary and verifies every input as tracked, present in the base tree, regular, non-symlink, and identical across expected, base-tree, and filesystem SHA256 values before parsing frozen bytes. The Exact6 runtime issue inventory is excluded; the authoritative issue predecessor is the committed ADMIT_007 adapter-design Exact11 inventory. The checker independently freezes the Exact18 paths and SHA256 values, output names, CSV headers, complete 56/107/15/36/11 semantics, the full Exact98 manifest, and Exact37 readiness. The production handler path remains pure in memory and does not import the adapter design gate or reach metadata I/O.

Readiness is limited to the implemented Exact7 synthetic single-rule runtime and the next audit, `audit_covapie_admit_008_formal_evaluator_interface_preconditions_v1`. It does not claim ADMIT_008 implementation, all-rule coverage, provider mapping, real candidate evaluation, bulk-download readiness, or training readiness. A feature-semantics audit remains mandatory before training; Step12D was only a smoke legality check, not the final training-feature contract.
